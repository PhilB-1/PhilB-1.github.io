SQL Cheat Sheet

In Terminal

  • type sqlite3 to work in a database
  • type .exit to exit
  • type .schema to view the schema
  • type .help for help

Data Types

  • Integer, a positive or negative whole number
  • Text, a text string
  • Date, the date formatted as YYYY-MM-DD for the year, month, and day
  • Real, a decimal value

Clauses

Commands. They are written in all caps, and perform specific task in SQL
  • Ex: CREATE TABLE celebs (id INTEGER, name TEXT, age INTEGER);
  • INSERT INTO celebs (id, name, age) VALUES (1, 'Justin Bieber', 21); - Inserts a row
  • SELECT * FROM celebs; - lets you view the entire table
    • Replace * with the column name to only see that column
    • You can query more than one column by separating the names with a comma.
    • DISTINCT genre - queries the unique items in that column
  • UPDATE celebs
  • SET age = 22 WHERE id = 1;
  • ALTER TABLE celebs ADD COLUMN twitter_handle TEXT;
  • DELETE FROM celebs WHERE twitter_handle IS NULL; - will delete all rows where the condition is met, in this case null in the twitter_handle column
  • SELECT * FROM movies WHERE name LIKE 'Se_en'; - LIKE is an operator used with the WHERE clause to find a certain bit of information. LIKE is not case sensitive. (_) is a wildcard that allows you to search for a word with any character in the _ place.
  • SELECT * FROM movies WHERE name LIKE '%man%'; - % is another operator. When surrounding a word searches for items containing those characters
  • SELECT * FROM movies WHERE name LIKE 'a%'; - When % is at the end, it search for items starting with those characters. %a would search for items ending with a.
  • SELECT * FROM movies WHERE name BETWEEN 'A' AND 'J'; - BETWEEN is an operator that queries all items that start with the first character and up to, but not including the second character.
  • SELECT * FROM movies WHERE year BETWEEN 1990 AND 2000 AND genre = 'comedy'; - AND is used to add another condition when querying
  • SELECT * FROM movies WHERE genre = 'comedy' OR year < 1980; - OR is used to include more than one conditions in a query
  • SELECT * FROM movies ORDER BY imdb_rating DESC; - ORDER BY lets you order your content by DESC for descending or ASC for ascending
  • SELECT * FROM movies ORDER BY imdb_rating ASC LIMIT 3; - LIMIT returns only that many queries

Operators

Commonly used when using the WHERE clause
  • = equals
  • != not equals
  • > greater than
  • < less than
  • >= greater than or equal to
  • <= less than or equal to
