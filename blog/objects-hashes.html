Ruby Classes

February 14th, 2016

When starting to interact with and learn javascript I began to notice a lot of similarities. What was even more interesting were the difference some of the similarities have. Most often in syntax, but even in how the function. This can be most easily noticed with objects in javascript and hashes in ruby. Below you can see an object and a hash with identical contents. As you can see they look almost exactly the same. If you used symbol notation with the ruby hash, the only difference would be the syntax var at the beginning of the javascript object.

var sample = { first: 1, second: 2, third: 3, }
sample = { first: 1 second: 2 third: 3 }

Now the first difference we will talk about will the how to access the values associated with keys(ruby) and properties(javascript). In ruby you use brackets [] to access something in the hash, for example sample[first] = 1. In javascript you use a dot (.), for example sample.first = 1. Another convenient similarity is how to add new keys or properties to your hash or object. You can do it explicitly by simply type sample[fourth] = 4 or sample.fourth = 4. With this you can also nest arrays or other hashes/objects within you hash/object. One of the main differences between objects and hashes is that you can store functions within your object that will be called on that object allowing you to manipulate it in some fashion. You cannot store methods inside of a hash in ruby. You can store a hash in a method to manipulate the information stored there, but that is it.
