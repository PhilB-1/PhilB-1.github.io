Ruby Classes

February 7th, 2016

Ruby is a object oriented programing language. This means you can organize your methods and what is effected using classes and objects. Classes have the ability to hold multiple methods. Using classes you can create an instance of the class, an object, and use the methods to only affect that object. Let's do a simple example. We'll write a simple class person and initialize it with a couple variables and create a puts statement in another method.

  1. class Person
  2. def initialize
  3. @name = "Phillip"
  4. @city = "Oakland"
  5. end
  6. def intro
  7. puts "My name is #{@name} and I live in #{@city}.
  8. end
  9. end
  10. phillip = Person.new
  11. introduction = phillip.intro

In the code written above we see a few things happening. First we used the initialize method to establish some variables with an @ sign in front of the name. The @ sign denotes a instane variable. Usually we've seen variables that are local and can only be used in the method they are created in. These instance variables can be used across the object in the different methods it contains. If we go to the bottom of the code we can see us call the initialize method of the Person class using Person.new. This creates a new object of Person "phillip." This allows us to now call the intro method on the object "phillip" using phillip.intro. We could also create as many objects of the class person as we want. We would probably want to take arguements if we did this. If you look at the code below, you will see our old code now taking arguments.

  1. class Person
  2. def initialize(name, city)
  3. @name = name
  4. @city = city
  5. end
  6. def intro
  7. puts "My name is #{@name} and I live in #{@city}.
  8. end
  9. end
  10. phillip = Person.new("Phillip", "Oakland")
  11. introduction_phil = phillip.intro
  12. elisabeth = Person.new("Elisabeth", "Oakland")
  13. introduction_elis = elisabeth.intro

We now see our instance variables @name and @city set equal to the arguments name and city so that they can now be used across the object. We also see two objects created by the Person class, a phillip object and an elisabeth object. These classes allow us to organize our code and more easily classify what methods are effecting it.
