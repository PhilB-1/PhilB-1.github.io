Understanding the Command Line and Getting Git

January 1, 2016

I know for many people the command line can be a confusing and scary thing. It was for me when I first time I saw it, and sometimes can still be as I am still learning and figuring things out. It's completely different from the interactive interface you've become accustomed to. I remember the first time I opened terminal and typed in all these weird codes to unlock and root my phone. I had know idea what I was typing in and was praying the instructions I was following were right. Now that I am a bit more familiar I understand how similar it is to what I've done before and how it all relates, and soon enough, so will you.

The command line is the most basic and fundamental way to interact with your computer. There's no windows, tabs, or a mouse to use, just your keyboard and commands. While it can be intimidating this makes navigating and working in your system much faster and more effecient. The great thing is that you navigate using the same file system you've already been accustomed to, just without all the pretty design. Another name for the command line is the shell which you use commands to navigate. Commands tell the shell what task to perform. I believe the most important commands to be cd, pwd, mkdir, ls, grep, vi, cat, and man. Using the shell is foundationally navigating through directories which means you must know how to navigate through them. Thats why cd, pwd, and mkdir are so important because to need to be able to navigate to a directory, know what directory you are in, and create a new directory if needed. In the same idea, ls is important because you will always need to see what files and directories are within the directory you are using or plan to use so you can navigate and work with the correct file. When working in an environment without a robust and polished user interface search is very valuable, thus the grep command being so useful. Once a file is found, you will need to see what the contents are and using the cat command to print them out is a great way to do that. With the shell being all about efficiency the vi command is a necessity. With most of our work being creating code to be run by a program, it is invaluable to have an editor right within the shell without having to exit and open another program to create or edit a file. And lastly man, which will give you detailed information about every command. Often times we will not remember exactly what a command does or what are all the options that accompany it, but this command allows us to easily access that information.

Now that you know a little about navigating the command line we can talk about git. Git it used to do version control on a project, in this case code, you may be working on. Version control allows you to easily keep track of changes on the same file/files while also allowing you to easily go back to a previous version. This is even more important when you are collaborating with others. This helps for communicating changes and allows all parties to see what is happening. Think of using Google Docs and how it tracks the changes for a document. The difference is that git will allow you to do this with any file. So what exactly is git? Git is an amazing piece of software that lives locally on your computer and accessed through the command line. Git allows for version control of a project and can be used with anything from photoshop projects to word documents, but is generally used to track changes in code. Git is very powerful, but also very simplistic and very fast, which is one of the reasons it has become so popular with so many developers. Different versions are tracked through commits, which records all the changes made to a project and allows the person making the changes to add a description of what is being done. This allows developers to work on different parts of a project by copying the commit and working on a new branch and merging them back to the master branch once their portion is complete. This allows each developer to make changes while simultaneously seeing each other's changes. It also allows you to easily go back and find mistakes or revert to a less buggy version should problems arise. Thatâ€™s why it is important to commit often. While git lives locally on your computer, GitHub allows you to host a repository on the internet for better collaboration and immediate access to the repository of a project by all members of a development team. This allows you to merge and pull a version without having to know whether another developer's machine is on and able to be accessed. You will still need to utilize git to perform all of your actions, but know you can manage a copy of the repository that is hosted on GitHub.

I hope this has been a helpful introduction to the command line and git, and you are now anxious to embark on your own coding journey.
